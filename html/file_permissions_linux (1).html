File Permissions in Linux

Project Overview

This project demonstrates how file and directory permissions were analyzed and updated to enhance system security. Using Linux commands, such as ls -la and chmod, permissions were tailored to organizational needs.

Command Examples

# Check file permissions
ls -la projects/

# Remove write access for others on project_k.txt
chmod o-w project_k.txt

# Update permissions for hidden files
chmod u-w,g-w,g+r .project_x.txt

Explanation

Steps included:

  • Checking current permissions using ls -la to identify unauthorized access.
  • Modifying permissions with chmod to remove or add access for specific user groups.

Results

The following outcomes were achieved:

  • Unauthorized write access was removed from sensitive files.
  • Permissions were optimized to align with security policies.

Impact

By updating file permissions, the organization reduced its risk of data breaches and ensured compliance with security standards.
