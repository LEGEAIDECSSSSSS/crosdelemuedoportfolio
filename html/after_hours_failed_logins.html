After-Hours Failed Login Attempts

Project Overview

This project focused on identifying potential security incidents by filtering failed login attempts that occurred after business hours (after 18:00). Using SQL queries, I analyzed login data to pinpoint suspicious activities.

SQL Query

SELECT *
FROM log_in_attempts
WHERE login_time > '18:00'
  AND success = FALSE;

Explanation

The query retrieves all records from the log_in_attempts table where:

  • The login_time is after 18:00.
  • The success column is marked as FALSE (indicating failed attempts).

This helps narrow down the data to investigate potential after-hours breaches.

Results

The query returned multiple failed login attempts that occurred after 18:00. These results were further analyzed to identify patterns and potential unauthorized access.

Impact

This analysis enabled the organization to:

  • Identify unusual login patterns after hours.
  • Implement stricter access controls for off-hour activities.
